      6. Data Gathering

      Our dataset originally contained 10 unique CSV files and 10 unique JSON files for 10 regions. Let's start with the SQL process. I'll be brief!

      SQL

      We used POSTGRESQL to load and join our table. Firstly, we needed to get our JSON and CSV data into PostgreSQL. JSON stands for JavaScript Object Notation. Here are the steps that we took:

      1. Create Tables
      2. Insert JSON data
      3. Insert CSV files using pgAdmin
      4. Query JSON file
      5. Applied aggregate functions to JSON data

      Unfortunately, during this process, we found that we could not use 4 of the 10 JSON files. Since the data was unable to be used, we modified our project to only include 6 regions.

      We created 2 major tables and cleaned up for the visualizations to pull data that's appropriate for the maps. This was loaded into tooltip. We reverted the code to our original version with just all stats. This code included FLASK, GEO JSON, and querying the database.

      Sentiment Analysis

      This is the fun part. Sentiment Analysis is also known as "Opinion Mining". Why choose to perform a sentiment analysis? The goal was to understand the attitudes, opinion, and possibly emotions expressed by YouTube users.

      We embarked on this journey to see if we could possibly understand our users truly feel about the most viewed YouTube videos.

      Using a python, our analysis was completed on the qualitative tags included in the CSV and JSON files. We created box plots, scatter plots, covariance and correlation graphs, and polarity distribution graphs. We then took this data and created some wordclouds!
